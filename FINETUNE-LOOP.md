# Eval-driven fine-tune → agent loop

Как из **датасета** получить **дообученную модель** и **агентного чат-бота**, доказав на каждом
шаге, что стало лучше (а не «по ощущениям»). Eval-лаба — это **измерительная половина** этого цикла.

## Три режима тестирования (что каждый доказывает)
| Режим | Вопрос | Где в лабе |
|---|---|---|
| **Dataset testing** | «Насколько хороши ответы на МОИХ данных?» | Run / Results (jury + verifier) |
| **Model testing** | «Модель A лучше B на моих данных — реально или это шум?» | Compare + `stats.py` (парные тесты, IMPROVE/FLAT/REGRESS по нижней границе CI) |
| **Agent harness + model** | «Ведёт ли модель себя верно КАК АГЕНТ — правильный путь tool-ов, безопасно, доводит юзера до цели?» | Trajectory / Injection / Sim-user |

Ключ: **dataset** меряет ответы, **model** доказывает разницу между моделями, **agent** тестит то,
что реально отгружаешь в прод (агент), а не одиночные ответы.

## Полный цикл (твой кейс: «есть датасет, хочу дообучить»)

**0. Сплит датасета.** Разбей на `train` (на нём модель учится) и `test` (held-out, модель его
НИКОГДА не видит). Эвалить только на `test` — иначе меришь зубрёжку, а не обобщение. Для дообучения
нужны пары `(инструкция/вход → желаемый ответ)`, желательно сотни-тысячи.

**1. Baseline (ДО).** Прогон эвала на `test` с **базовой** моделью → «число до». Без него нельзя
доказать, что дообучение помогло. [лаба: Run → Results]

**2. Дообучение (fine-tune).** Учим LoRA на `train`.
⚠️ **Грабля железа:** локальная GPU — **AMD RX 6700 XT (Vulkan)**, а Unsloth требует **NVIDIA/CUDA**.
Значит дообучаем на **бесплатном Google Colab T4** (всё ещё $0) или Kaggle. Выход: GGUF-файл →
скачиваем → грузим в LM Studio. (Локальный фолбэк: `llama.cpp finetune` — медленно и ограниченно.)

**3. Eval дообученной (ПОСЛЕ).** Тот же эвал, тот же `test`, новая модель → «число после».

**4. Доказать строго.** Compare: base vs fine-tuned на ОДНИХ И ТЕХ ЖЕ кейсах, парный тест →
**IMPROVE / FLAT / REGRESS** по нижней границе 95% CI. Это и есть уникальная ценность: не «вроде
лучше», а **доказано, что прирост реален, а не шум судьи**. [лаба: Compare + `stats.py`]

**5. Agent harness (если строим агента, а не Q&A-модель).** Оборачиваем дообученную модель в
**PydanticAI** (инструменты + system prompt + цикл агента), гоняем Trajectory (верный ли путь
tool-ов) + Injection (безопасно?) + Sim-user (LLM-персона доходит до цели?). Тестим АГЕНТА.

**6. Gate + ship.** CI-gate (движок #7) пинит baseline; катим только если нет регрессии. Прошёл —
агент становится продуктом-чат-ботом → подключается в AutoEvalsBots как «мозг» канала.

## Почему это «уникальные реальные результаты»
Дообучить может любой. Уникальность — что ты **доказываешь** прирост на СВОИХ данных строгим
бесплатным жюри (разные семейства) + статистикой + агентной траекторией + проверкой безопасности,
и **гейтишь** каждое изменение. Это и есть «eval-driven AI engineering» — сильная история портфолио
и правильный способ родить агентного чат-бота на дообученной LLM «с рождения».

## Что достроить в дашборде (rebuild → «Fine-tune Loop»)
Новый режим, ведущий по шагам 0→6 наглядно:
1. Загрузка датасета + авто-сплит train/test.
2. Кнопка Baseline (эвал базовой на test, сохранить «до»).
3. Fine-tune: генератор **Unsloth Colab-ноутбука** (раз локально AMD не тянет) + регистрация
   полученного GGUF в ростер.
4. Eval дообученной (тот же test).
5. Before/After с вердиктом stats (переиспользуем Compare).
6. Agent eval (PydanticAI) + gate-вердикт.

Переиспользует движки, что уже есть. Новое: UI сплита, генератор ноутбука, регистрация модели,
before/after-вид, обёртка-агент.

## Открытые решения (см. чат)
- Где дообучать: Colab T4 (реком.) / llama.cpp локально / своя NVIDIA-GPU.
- Цель: single-turn модель или агентный чат-бот (или оба).
- Объём первого инкремента rebuild.
