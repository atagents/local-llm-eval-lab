# FreeStack Evals — Eval-Driven AI Engineering, $0 and local

A working lab for **measuring AI systems honestly** — models, agents, and fine-tunes — on a
**100% free, local stack** (LM Studio + free OpenRouter judges, no data leaves the machine).
It is both a toolkit of evaluation engines and a demonstrated *method*: don't trust a change,
**measure it** — and ship only what the numbers (with statistics) say is real.

> Thesis: the moat in AI products isn't the model — it's the rigor that proves a change helped.
> Anyone can fine-tune or tweak a prompt; few can show, with a held-out test and a confidence
> interval, that it actually got better and didn't quietly regress.

---

## Flagship: an eval-driven fine-tune loop (worked end-to-end, every step measured)

Goal: build a smart, warm companion agent ("Mira") and **prove** each step instead of trusting it.
The eval lab caught two regressions and found the real lever — none of it on vibes.

**Leaderboard on a hard held-out test (`companion_hard`, neutral judge), persona-fit / voice-match / failure-rate:**

| Variant | persona-fit | voice-match | failures | verdict |
|---|---|---|---|---|
| local 7B base + prompt | 3.50 | 2.42 | 4% | reference |
| local 7B as an agent | 3.42 | 2.38 | 17% | **FLAT** — prompt plateaued |
| **fine-tune v1** (cheap synthetic) | 2.54 | 2.46 | **62%** | **REGRESS** — overfit to one trait |
| **fine-tune v2** (rebalanced) | 2.29 | 2.29 | **67%** | **REGRESS** — overfit to bland empathy |
| **agent on a strong cloud brain (DeepSeek)** | **4.04** | **3.71** | 4% | **IMPROVE** (+1.29 voice, far beyond noise) |
| v3 fine-tune (402 frontier-curated pairs) | *measuring* | | | data-quality fix |

**What the measurement proved (not assumed):**
- The **mean** was the wrong target (near ceiling) → we measured **failure-rate, voice-match, and
  no-prompt robustness** — the things a fine-tune is supposed to move.
- Easy cases + a lenient judge = a **ceiling effect** (everything scores 5.0, nothing is
  distinguishable) → we built a **hard test** + a **strict rubric** so scores spread.
- A **+0.1 gap is noise** — verdicts gate on a paired confidence-interval lower bound, not the point estimate.
- **Fine-tuning on cheap synthetic data regressed twice** — the eval caught it; we did not ship a
  worse model. The bottleneck was **data quality**, not hyperparameters.
- The real levers, ranked by what the numbers showed: **data quality > model/brain >
  prompt/agent engineering** (prompts and the agent layer plateaued).
- v3 data was then **generated by a frontier model and strictly verified** (a 32-agent workflow:
  16 scenario buckets → write → adversarial keep-only-excellent → dedup + leakage-check vs the test set).

## The evaluation engines

**Per-answer / per-model metrics** (against any OpenAI-compatible endpoint, $0 local):

| Engine | What it answers | Where |
|---|---|---|
| **Jury** (ours) | "Is this answer good?" — 5-family LLM-as-judge ensemble, median, no reference | `jury/` |
| **Objective verifier** (ours) | "Does the code actually pass tests?" — sandboxed pytest, ground truth | `evals/verifier/` |
| promptfoo | "Did quality regress? which prompt wins?" | `evals/promptfoo-demo/` |
| DeepEval | "Does it pass our metric gate?" (G-Eval as pytest) | `evals/deepeval-demo/` |
| garak | "Can it be jailbroken / leak data?" (red-team) | `evals/garak-demo/` |
| lm-eval-harness | "What's the MMLU/GSM8K number?" | `evals/lm-eval-demo/` |
| RAGAS | "Is the answer grounded, not hallucinated?" | `evals/ragas-demo/` |

**Agentic engines** — score the *behaviour* of an agent, not just a single answer:

| Engine | What it answers | Where |
|---|---|---|
| **Trajectory** | "Did the agent take a legal path — right tools, valid args, reached the goal?" (no LLM, deterministic) | `evals/trajectory-eval/` |
| **Statistical rigor** | "Is this delta real or noise?" — BCa bootstrap CI + paired tests, ship on the lower bound | `dashboard/stats.py` |
| **Prompt-injection gate** | "Can the agent be hijacked?" — honeypot trip-log → deterministic attack-success-rate | `evals/injection-gate/` |
| **Simulated user** | "Does an LLM persona driving the agent get a good outcome?" | `evals/sim-user/` |
| **Judge-bias** | "Is our judge biased?" — position / verbosity / sycophancy probes | `evals/judge-bias/` |
| **Failure taxonomy** | "What kinds of failures, and how reliably labelled?" — Cohen's kappa | `evals/failure-taxonomy/` |
| **Regression CI-gate** | "Did this change regress vs a pinned baseline?" — exit≠0 on regression | `evals/ci-gate/` |

Full ecosystem catalog (frameworks, licenses, install): [EVALS-CATALOG.md](EVALS-CATALOG.md).

## The dashboard

A Streamlit eval console (the interview-facing showcase): CRUD test cases, run jury / correctness /
faithfulness / verifier / **the fine-tune loop** against local models, with tables, charts, paired
stats, and history. Dark-branded, password-gated, runs locally; demoed at **evals.telarlabs.online**.

```bash
source env-wsl.sh && source ~/.config/freestack/keys.env
dashboard/.venv/bin/streamlit run dashboard/app.py --server.address 127.0.0.1
```

The 🎓 **Fine-tune** tab walks the whole loop: dataset → train/test split → baseline → a generated
Unsloth Colab notebook → eval the result → before/after with statistics.

## The companion agent

`companion/mira_agent.py` — a **PydanticAI** agent (`pydantic-ai-slim[openai]`), provider-configurable:
brain = local Qwen in LM Studio for dev ($0), or a strong cloud model (DeepSeek / OpenRouter) for
quality, via the same `OpenAIChatModel` transport. Persona + conversation memory + a `remember` tool.

```bash
source env-wsl.sh && companion/.venv/bin/python companion/mira_agent.py            # local Qwen, $0
# strong brain:
export MIRA_BASE_URL=https://api.deepseek.com/v1 MIRA_API_KEY=$DEEPSEEK_API_KEY MIRA_MODEL=deepseek-chat
```

## Architecture & stack ($0)

- **Models** run in LM Studio on the host GPU; evals reach it over an OpenAI-compatible endpoint.
  The WSL subtlety (host IP, open auth) is handled by [`env-wsl.sh`](env-wsl.sh) — `source` it first.
- **Judges** can be local (free) or offloaded to free OpenRouter / DeepSeek for speed — **judge ≠
  model-under-test** (no self-family bias), juries drawn from different families.
- **Code-executing evals run in a Docker sandbox** (`--network none`) — model-generated code never
  touches the host.
- **Fine-tuning** runs on a free Google Colab T4 (the local GPU is AMD/Vulkan; Unsloth needs CUDA);
  the dashboard generates the notebook. Inference stays local.
- Each component has its own venv (heavy deps stay out of git).
