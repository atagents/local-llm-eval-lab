# Jury - single judge vs 5-family ensemble (2026-06-23)

Task: **summary faithfulness** (binary FAITHFUL/UNFAITHFUL), **15** hand-labeled items (7 FAITHFUL / 8 UNFAITHFUL). Judges run sequentially (12 GB VRAM, one model at a time). Honest run - gold set authored as a fair benchmark, not tuned. Majority vote (not accuracy-weighted: weighting by accuracy on the same set is circular).

## Headline

- **Single judge (phi-4-14b)**: agreement 14/15 (93%)
- **5-family jury (majority)**: agreement 15/15 (100%)
- **Best single model**: ministral-3-14b-reasoning 15/15 (100%)
- Delta jury - single: **+1** items
- Jury vs single beyond cost: jury used **10966 tokens / 515.3s** vs single **2244 tokens / 165.0s** (~4.9x tokens, ~3.1x latency).

## Reading the result

- Heterogeneity HELPED on items where single was wrong but the jury was right: ['s2'].
- Heterogeneity HURT on items where single was right but the jury was wrong: none.
- The jury costs ~5x tokens and ~5x latency (5 models, sequential JIT loads). It is worth it only when the agreement gain offsets that, i.e. on subtle/ambiguous items where single-model errors are de-correlated across families. On clear-cut items a single good judge matches the panel at 1/5 the cost.
